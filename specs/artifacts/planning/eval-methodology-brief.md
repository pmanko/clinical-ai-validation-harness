# Evaluation methodology — design decisions & research foundations

> A brief on how we measure answer quality in this harness, what the research supports, and which design decisions have the strongest foundation vs which are judgment calls. Validated against the repo's own settled methodology (006 spec + `guardrails-methodology-research.md`) and ~25 research papers (arXiv) gathered 2026-06-04. Citations are arXiv IDs.

## TL;DR — strongest to weakest foundation

1. **Decomposed clinical rubric** (accuracy/completeness/relevance + abstention + citation_groundedness + harm) — *strongest*; maps 1:1 onto the published clinical-eval consensus.
2. **Atomic-claim faithfulness** (decompose → verify each against the chart) — strong, human-validated; the right backbone for "fabricated facts/trends".
3. **Two-layer citation check** (deterministic resolve + semantic support) — strong; the two layers catch different, complementary failures.
4. **Abstention as a categorical outcome** — strong; the axis most legacy rubrics omit.
5. **Strong-LLM-as-judge, pointwise vs ground truth, rubric+CoT** — well-founded *for this config*, with two binding caveats (clinical-agreement cap, fluency confound) → must be human-calibrated, advisory-not-gating.
6. **N=21, one patient, one judge, scored once** — *weakest*; a sampling + single-rater limitation, not a methodology. Needs more patients, a jury, and human calibration.

## Decision-by-decision

### 1. Exercise the real chartsearchai pipeline, multi-arm side-by-side
Driving the real `/chat` (chart build → prompt → `{answer,citations,blocks}`), not raw model output (006 spec). **Foundation: strong** — it's the correct unit of analysis (the system, not the model). No caveat beyond: the chart build + prompt are themselves variables (hence the enforcement map + harmonization work).

### 2. Deterministic, no-LLM metrics first (latency, json_valid, citation_count, abstained)
006 FR-006.4. **Foundation: strong** — objective, cheap, no judge bias; the repo wisely makes these the floor and treats the LLM judge as the *added* layer.

### 3. Decomposed clinical rubric (the Scout rubric)
accuracy / completeness / relevance (0–10) + abstention_outcome + citation_groundedness + harm. **Foundation: strongest.** Accuracy, completeness, and harm recur identically across **Med-PaLM's 12-axis physician rubric [2212.13138]**, **Med-PaLM 2 [2305.09617]**, **MedHELM [2505.23802]**, and **HealthBench [2505.08775]**. Completeness is independently justified as *distinct* from accuracy (a model can be ~90% accurate yet omit clinically important info in ~47% of summaries — npj Digit. Med. 2025). Harm has a concrete scale (AHRQ severity × likelihood, [2212.13138]). **Caveats:** *relevance* is the weakest-defined axis — the literature splits it into commission (inappropriate content) vs instruction-following vs context-awareness, so a single score conflates three behaviors; *harm* is the noisiest axis (physicians disagree on severity — needs anchored examples / multiple raters); the rubric omits **bias/demographic applicability** and **clarity/communication**, both standing axes in Med-PaLM/HealthBench — add or consciously scope out.

### 4. Atomic-claim faithfulness (decompose → verify each against the chart)
The "fabricated facts / invalid trends" measure. **Foundation: strong, human-validated.** This is the established standard: **FActScore [2305.14251]** (atomic facts, % supported; estimator within <2% of human), **RAGAS faithfulness [2309.15217]** (claims inferable from the *retrieved context* — 95% pairwise human agreement on WikiEval), **SAFE [2403.18802]** (72% per-fact agreement, >20× cheaper than humans). Our setup is closest to **RAGAS-against-the-chart-as-closed-context** (we *have* the ground-truth source). **Caveats:** (a) **precision ≠ completeness** — a faithful answer that omits a critical lab scores perfectly, so faithfulness must be paired with the completeness axis (FActScore's own warning); (b) the 72–95% are open-domain — clinical entailment (negation, temporality, units, reference ranges) is materially harder, so the verifier needs its own clinical validation; (c) **faithful-to-chart ≠ clinically correct** (the chart can be stale); (d) **numeric/relational fabrications** — the exact "invented second value / trend from one point" risk — are the hardest: plain NLI under-catches them, and a trend often doesn't survive atomic decomposition cleanly (verifying it means reasoning over an *absence*). Mitigation: claims must preserve numbers + relations, do explicit numeric-entailment, and consider graded severity over binary supported/not [2410.12222].

### 5. Two-layer citation check (deterministic resolve + semantic support)
**Foundation: strong; the layers are complementary, not interchangeable** [ALCE 2305.14627, AIS 2112.12870, RARR 2210.08726, AttrScore 2305.06311, reference-hallucination 2604.03173]. Layer 1 (deterministic): does each cited record's `resourceUuid` resolve to a real chart record that was in the retrieved set — the *only* sound detector of a **fabricated citation** (NLI metrics can't tell a fabricated index from a merely-unsupported one). Layer 2 (semantic): ALCE-style citation recall/precision via NLI — the actual *groundedness* measure. **Caveats:** chartsearchai **echoes** cited `[N]` into `references`, so an index-in-array check is useless — Layer 1 must be uuid-resolves-against-chart (verified this session). Layer-2 NLI (e.g. TRUE/T5-11B) is general-domain and degrades OOD [AttributionBench 2402.15089] → validate against human AIS judgments; adopt AttrScore's **Extrapolatory vs Contradictory** split ("chart doesn't mention this" ≠ "chart contradicts this" — clinically very different).

### 6. Abstention as a categorical outcome (correct / over-abstained / failed-to-abstain)
**Foundation: strong** [AbstentionBench 2506.09038; Kamath 2006.09462; Mallen 2212.10511]. A confusion matrix over {answerable?}×{abstained?}: failed-to-abstain (hallucinated when it should have abstained) lowers abstention recall; over-abstention lowers precision. The clinically meaningful "honest absence" = answering "not documented" when X isn't in the chart; substantive answers there are a faithfulness violation, not just an abstention miss. Most legacy clinical rubrics omit this — including it is a strength.

### 7. Strong-LLM-as-judge, pointwise vs ground truth, decomposed rubric + CoT
**Foundation: well-founded for THIS configuration** [G-Eval 2303.16634 — CoT+form-filling rubric materially improves human alignment; rubric+CoT is the single best debiasing config, 2604.23178]. Pointwise scoring against a known chart **structurally neutralizes** the headline pairwise biases (no A/B ordering → position bias N/A; verbosity is second-order for strong judges). Using an Opus-class judge is the standard choice; the repo's own research says local 4–8B verifiers are unreliable → escalate to the cloud model (= "judge = you"). **Two binding caveats:** (a) **expert-clinical agreement caps below the ~80% general headline** — in a global-health clinical study the best judge (Claude-4.1-Opus) matched clinicians on only 4/11 criteria and a jury barely moved it → any clinical-correctness claim must be **human-anchored** (calibrate against a physician-scored gold subset, report agreement, as LLMEval-Med [2506.04078] does); (b) the **perplexity/fluency confound** [2410.21819] survives reference-grounding — a confidently-worded-but-wrong answer can outscore a terse-correct one, directly hazardous for clinical accuracy. Secondary, conditional: **self-preference** [≈10% / 0.52 bias] is dormant only because the judged backends are Gemma/MedGemma; it re-enters if Opus ever judges Claude-generated (cloud-router) outputs → use a different judge family or a PoLL panel [2404.18796] for those runs. Posture: ship **advisory-only, calibrated**, never gating — which matches 006's human-only-v1 stance.

### 8. Sample: N=21 scenarios, one patient, one judge, single pass
**Foundation: none — this is the methodology's weak point.** It surfaced real failures (directionally useful), but: small sample, single patient (Zabella), single judge, no inter-rater/jury, no human calibration. It also had a process bug this session (a 1100-char dump truncation under-scored multi-turn answers, caught + corrected). **Mitigations, in priority:** (1) score against *human gold* on a subset and report agreement (the #1 lever per decision 7); (2) more patients (Aloice + others) for generalizability; (3) a small **jury** of disjoint-family judges [PoLL 2404.18796] to dilute single-model bias; (4) preserve full answers + claim-level provenance (no truncation).

## How this maps to the implementation
- The Scout rubric + the two-layer citation check + atomic-claim faithfulness become the shared `reconcile` module (plan Track 1), used by both the offline judge (Track 5) and the runtime validator (Track 4).
- The deterministic uuid-resolves check is the no-LLM fabrication catch (cheap, sound) and should run on every answer.
- The judge stays advisory + gets a human-calibration subset before any score is treated as authoritative.
