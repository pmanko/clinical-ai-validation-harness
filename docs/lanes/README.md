# Development lanes

One **dossier per active development lane** — the durable, branchable operating unit for each lane.
A dossier is what you read to cold-start a lane in its own worktree: branch
mechanics, scope, merge/pin gates, and the verbatim kickoff prompt.

## How the planning surfaces fit together

```
specs/roadmap.canvas.tsx     VISUAL   — milestone/feature dependency model + in-flight lane cards
        │
docs/dev-roadmap.md          INDEX    — north-star, launch sequence, the lane table (status + gates + links)
        │
docs/lanes/L?-*.md           DOSSIER  — how to run a lane (this directory): branch, gates, scope, kickoff
        │
specs/artifacts/planning/*-brief.md   BRIEF — technical design / Spec Kit input (only lanes that need one)
        │
specs/0NN-*/                 SPEC     — formal requirements, generated by /speckit-specify at kickoff
```

**Separation of concerns (so the layers don't duplicate):**
- **Dossier** = *operating* — branch/worktree/repo + branch-model, scope, merge gate, pin-bump gate,
  kickoff prompt, status, links down. The only place that carries branch/gate facts.
- **Brief** = *technical design* — the architecture/contracts that feed Spec Kit. Not every lane has one.
- **Spec** = *formal requirements* — generated; never hand-edited (edit the brief, regenerate the spec).
